linux系统，主要分debian系和redhat系，还有其它自由的发布版本。

1、debian系主要有Debian，Ubuntu，Mint等及其衍生版本；

2、redhat系主要有RedHat，Fedora，CentOs等，

关于Github上master

准备

在Github上新建了一个仓库
在本地使用git初始化了一个文件夹
一般情况下:
本地仓库为master
远程仓库分支为main^1

在ssh能够连接并没有错误的前提下

步骤

1、初始化一个仓库

  • git init

2、添加git config 用户信息

  • git config user.name <name>
  • git config user.email <email>

3、添加远程仓库源(路径)

  • git remote add origin git@github.com:<user>/<repository-name>

4、按照廖雪峰大神的Tech，完成了这些我首先就执行了以下命令关联仓库:

  • git push -u origin main

    但是它给我返回了错误信息，不是hint，也不是refusing或者rejected,就是单纯的error。

5、于是又试了很多次无用，只好根据提示来先拉取一遍再push

  • git pull origin main

    成功了
    于是我再push可无论是git push origin main,还是git push -u origin main还是一样的返回错误。

6、抱着试一试的态度我打出以下命令:

  • git push -u origin master

    没错，它成功了，但是竟然又创建了一个新的远程仓库的分支master^2

7、现在的情况就是本地分支:master
远程仓库中分支:main和master

  • git branch -a

查看所有分支

8、简单的思考了一下:在master改为main之前，无论是远端还是本地，默认的分支都是master，于是:

我查看了config 中 origin字段中的信息并将其所有的 master替换为了 main
再找到 head 指针中的 master 一并替换为main

  • git branch -d master删去分支

9、再进行git push操作已然成功！

Git使用记录

在我学习Git的期间，出现的一些问题以及关于这些问题的解决办法和想法

Git配置多账户ssh密钥

  • 问题

    我为两个github账号下的三个存储库分别创建了三个ssh密钥文件，且都放在了对应的每个存储库中的.ssh文件夹中，我以为这样它自己就能够找到私钥并匹配。之后可想而知无论怎么样也连接不上。

  • 解决办法

    首先先提到两个文件:
    位于user下的.ssh文件夹/user/.ssh/config
    和位于git安装目录下的/etc/ssh/ssh_config
    这两个配置文件前者是版本库的用户Host和ssh
    私钥路径指向的配置。
    后者是ssh配置，里面有一项为Identyfile的参数
    也是用来指向ssh私钥路径的。
    于是就可以使用这两个配置文件的参数修改来达
    到多账户配置
    值得注意的是，这是针对我这种把密钥都放在/user/.ssh路径(ssh-keygen生成密钥时的默认路
    径)之外的情况。

    而且，在我成功配置多账户之后，发现同一账
    户多存储库只有一个密钥能够生效。

对于无法拉取和推送到远程存储库

  • 检查origin是否配置好
    git remote add origin git@github.com:username/reponame

  • 配置好用户名和邮箱
    git config user.name xxx
    git config user.email xxx

  • 保证私钥路径指向正确的前提下如果无法推送可以先拉取同步一下。
    git pull origin master
